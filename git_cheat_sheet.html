Here we have an overview of the most popular functions in Git.

  • Installing Git
  • Creating a repository
  • Linking you local repository to GitHub
  • Adding files to the staging area
  • Commit to a repository
  • Commit to a remote repository
  • Status and log
  • Go back in time
  • Branching

Creating a Repository

If you want to create a repository of a project, place all the files if the project in the same folder. In you console, make this file your working directory. Once you are in that directory type:

$ git init

You'll see a message saying that a repository was created in your folder

Linking you local repository to GitHub

if you want to link your local repository to a remote repo, first you have to create your remote repository in GitHub. Then you can type in your terminal

$ git remote add origin http://github.com/username/repository_name

Adding files to the staging area

If you want to Git to keep track of a file, you need to add it to the staging area. To accomplish that, type:

$ git add filename

Also you can add all the files in your folder typing

$ git add .

Commit to a repository

A commit is like a snapshop of the files you have in the staging area in a particular moment. In order to commit to your local repository type:

$ git commit -m "Message"

Remember you have to add the files to the staging area everytime you want to do a commit. There's an option to commit adding all the files to the staging area

$ git commit -a -m "Message"

Status and log

Anytime you can check the status and git log

$ git status

The status will tell you in which branch are you in and if there's any chango to commit.
The git log command will display all the commits for your repository in the current branch you are

$ git log

You'll see a list of all the commits you've made

Go back in time

You can check the status of you code in a previous state (a previous commit) with the checkout command

$ git chechout 234ebc2345

If you look at your files, the will look like they were in the moments you committed to you repo.
if you wan to go back to the last state type

$ git checkout master
